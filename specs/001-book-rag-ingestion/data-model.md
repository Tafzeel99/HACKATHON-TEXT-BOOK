# Data Model: Content-to-Vector Pipeline

## Entities

### Document Chunk
- **id**: string (UUID) - Unique identifier for the chunk
- **text**: string - The actual text content of the chunk
- **url**: string - Source URL where the content was extracted from
- **timestamp**: float - Unix timestamp of when the chunk was processed
- **embedding**: list[float] - Vector representation of the text content

### Embedding Vector
- **vector_id**: string - ID that matches the Document Chunk ID
- **vector**: list[float] - High-dimensional vector representation (generated by Cohere)
- **collection_name**: string - Name of the Qdrant collection ("rag_embedding")
- **payload**: dict - Contains text, url, and timestamp from Document Chunk

### Vector Record (Qdrant Point)
- **id**: string - Unique identifier for the Qdrant point
- **vector**: list[float] - The embedding vector
- **payload**: dict
  - text: string - Original text content
  - url: string - Source URL
  - timestamp: float - Processing timestamp

## Relationships
- Each Document Chunk corresponds to exactly one Embedding Vector
- Each Embedding Vector is stored as one Vector Record in Qdrant
- Multiple Document Chunks can originate from the same source URL

## Validation Rules
- Text content must not be empty
- URL must be a valid URL format
- Embedding vector must have consistent dimensions (determined by Cohere model)
- Timestamp must be a valid Unix timestamp
